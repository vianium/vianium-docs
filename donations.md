# Donations & funding transparency

> Last updated: 2026-05-19

The Vianium ecosystem is developed and maintained by
[Angel Careaga](https://angelcareaga.com) as a personal open-source
effort. There is no company, no investors, and no roadmap obligation
attached to any donation. This page documents the channels you can
support the project through and how the funds are used.

If any of this changes — for instance, a sponsoring entity steps in or
the project incorporates — the change will be reflected here first and
in the [vianium](https://github.com/vianium) org profile.

---

## Donation channels

### Recurring or one-time, hosted by GitHub

**[github.com/sponsors/AngelCareaga](https://github.com/sponsors/vianium)**

The canonical channel. Recurring tiers + one-off pledges. GitHub Sponsors
handles payment processing and tax forms; no fees are passed to the
sponsor. Each individual repo's "Sponsor" button surfaces this link
through [`.github/FUNDING.yml`](https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/displaying-a-sponsor-button-in-your-repository).

### One-time tip, no account needed

**[buymeacoffee.com/soyangelcareaga](https://www.buymeacoffee.com/soyangelcareaga)**

Lighter-weight than Sponsors — useful if you want to drop a small tip
without creating a recurring relationship. Buy Me a Coffee retains
~5% as a platform fee.

### Commercial inquiries, consulting, custom builds

**hello@angelcareaga.com** · **[angelcareaga.com](https://angelcareaga.com)**

For organisations that want a commercial license over the Tier 3
(PolyForm-Noncommercial-1.0.0) products — vianigram, vianium-browser,
vianium-music, vianium-localsend — please reach out directly. The
foundation and domain libraries are already Apache-2.0 and have no
commercial restriction.

## What the funds are used for

In rough priority order, donations support:

1. **Time on the codebase.** Vianium is a deep specialisation
   (Windows Phone 8.1 + native MTProto + cross-DC routing + obfuscated
   transports). Each maintained component takes regular hours that
   would otherwise compete with paid work.
2. **Test hardware.** Windows Phone 8.1 devices across the RAM range
   for low-RAM validation, and the occasional Windows 10 Mobile build.
3. **CI minutes.** When the GitHub Actions free tier is exceeded
   (mostly for the matrix builds across ARM/Win32 × Debug/Release).
4. **Domain + hosting.** `angelcareaga.com`, `vianium.org` (planned),
   and any pages mirroring docs outside GitHub.
5. **Third-party tooling.** Code-signing certificates if the App
   store path is reactivated; static-analysis tooling licenses.

There is no "rainy day fund" — what comes in is reinvested in the next
quarter of work. If there is ever surplus beyond what these items
absorb, it is donated to upstream projects (TDLib, libopus, libwebp,
ffmpeg) and that will be noted in this file.

---

## What donations explicitly do NOT buy

- **Priority bug fixes** beyond what's already feasible.
- **Roadmap influence** — the roadmap is set by the project's own needs
  and the Tier 3 product users. Feature requests are welcome via the
  normal issue tracker but are not weighted by donation amount.
- **A commercial license** to Tier 3 products. Sponsoring is gratitude,
  not a contract; commercial use requires the dedicated channel above.
- **Affiliation or endorsement.** Vianium is not affiliated with
  Telegram, Microsoft, Google, or any other entity referenced in its
  code. Sponsoring does not grant any kind of co-maintainer status,
  voting rights, or implied partnership.
